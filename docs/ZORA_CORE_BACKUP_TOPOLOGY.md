# Zora-Core 3-Copy Backup Topology (Hardened)

This topology enforces encrypted-only storage with integrity validation.

## Backup Set Structure

Each backup set contains:

- `Zora-Core.zip.age`
- `SHA256SUMS.txt`
- `BACKUP_METADATA.md`

Generated by:

- `scripts/security/zora_core_encrypt_bundle.sh`

Validated by:

- `scripts/security/zora_core_restore_verify.sh`

## Storage Topology

- Copy 1 (Local Primary): encrypted backup bundle on local encrypted disk
- Copy 2 (Cloud Ciphertext): encrypted backup bundle in cloud object storage
- Copy 3 (Offline Air-Gap): encrypted backup bundle on offline external drive

All three copies are ciphertext; never upload or sync plaintext archive.

## Workflow

1. Generate encrypted bundle from current `Zora-Core.zip`.
2. Verify checksum immediately (`shasum -a 256 -c SHA256SUMS.txt`).
3. Replicate encrypted bundle to cloud and offline media.
4. Record operation in audit log.

## Verification Cadence

- Weekly: checksum-only verification on local and cloud copies.
- Monthly: full restore verification into temporary sandbox.
- Quarterly: key rotation plus fresh encrypted backup set.

## Minimal Verification Checklist

- [ ] Bundle exists in all 3 locations.
- [ ] `SHA256SUMS.txt` passes in all 3 locations.
- [ ] At least one monthly decrypt test succeeded.
- [ ] Recovery key location and access test confirmed.
