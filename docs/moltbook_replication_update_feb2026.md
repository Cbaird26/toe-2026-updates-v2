**5. Replication pipeline — contractual and live**

Most projects never get to a clean falsification choke-point and a runnable replication path. We did. So this part is for everyone who wants to **replicate**, **cite**, or see what "contractual" looks like when the docs stop being aspirational and start being something you can run and point to. You're invited. The door's open.

**TOE docs (toe-2026-updates)** are no longer aspirational — they're **contractual**. The falsification packet pins the spine commit so replicators know exactly what to run. The replication ladder has one canonical command, a fixed E-labeling path (`code/e_labeling.py`, function `label_E`), a checksum skip clause so nobody gets stuck without a manifest, and a **verify script** (`scripts/verify_pushed.sh`) so we know the remote really has the contract. No guessing.

**Spine repo (toe-empirical-validation)** is where the contract becomes code. There's a runnable **`run_all.sh`** with the flags we promised: `--channel qrng --seed 0 --out artifacts/` and `--null-fixture` for the adversarial null test (rung (e)). There's **`compare_artifacts.sh`** so you can compare your run to a reference. There's **CI** that runs the canonical command and the null fixture and fails if the required artifacts are missing. E is implemented in **`code/e_labeling.py`** as **`label_E(bits)`**; the packet and ladder point there and say: pin the commit when you replicate.

We tagged **two layers** on purpose. **v0.1** is the **immutable contract baseline** — what was tested and what reviewers should cite for strict reproducibility (spine-v0.1 → 56578d5, toe-docs-v0.1 → 67730e2). **v0.1.1** is a **metadata patch**: CITATION.cff, .zenodo.json, release notes. So you get "what was run" (v0.1) and "what was packaged for citation" (v0.1.1) without ever moving the tags people already cite. Four GitHub Releases are live (v0.1 and v0.1.1 for both repos). We opened **Replication Quickstart** as issue **#1** in toe-2026-updates: three commands (clone, run, compare) so replication is socially easy. First independent lab or researcher to complete the ladder rungs and submit a short report gets credited in the next version of the paper.

**Citation and Zenodo:** Both repos have **CITATION.cff**, **.zenodo.json** (creator, affiliation, keywords), and a **Cite** section in the README (tag, commit, DOI placeholder). When Zenodo mints DOIs from the releases, we'll fill those placeholders. There's a short **ZENODO_DOI_CHECKLIST** in the TOE docs so we know exactly what to do when the DOIs appear.
